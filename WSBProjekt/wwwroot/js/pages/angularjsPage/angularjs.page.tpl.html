Krótko o AngularJS

Otwarty framework oparty na języku JavaScript, wspierany i firmowany przez Google

Angular korzysta z wzorca MVC i promuje utrzymywanie słabych zależności pomiędzy warstwą logiki, prezentacji i danych.

Angular wprowadza tradycyjne serwerowe technologie, takie jak kontrolery zależne od widoku do aplikacji klienckich.

Poniżej postram się przedstawić główne cechy tego frameworku, na podstawie tego projektu.

Modularnść w AngularJS
angular.module('app', ['ui.router', 'ngAnimate'])

Moduły w AngularJS umożliwiają organizację kodu oraz możliwość dzielenia się nimi / uzywania w wielu aplikacjach.

Moduły deklarujemy tak jak jest to pokazane obok, na globalnym obiekcie angular w nawiasach kwadratowych umieszczamy moduły które chcemy "zaimportować"

Kontrolery, komponenty, usługi

Po utworzeniu modułu możemy tworzyc na nim rózne obiekty, które ułatwiają tworzenie i organizację aplikacji.

Zasada ich tworzenia jest podobna: w argumentach najpierw nazwa jaką nadajemy obiektowi, nastepnie tablica wstrzykiwana zależności (innych obiektów). ostatnim elemnetem tej tablicy jest funckja: właściwy obiekt.

Poniżej przedstawiam wybrane obiekty:

  • Kontroler
    angular.module('app')
    .controller('appController', ['$scope',
     function($scope){
       $scope.tplData = {
         shouldShowSidebar: true
       }
     }
    ])

    Kontroler udostępnia model oraz funkcje dla widoku. Wszystko, co przypiszemy na zaimportowanej usłudze $scope, będzie dostępne w widoku.

  • Komponent
    angular.module('app')
    .component('mainPage', {
       templateUrl: '/js/pages/mainPage/main.page.tpl.html',
       controller: 'mainPageController'
       }
     }
    ])

    Komponent to kawałek funkcjonalości połączony z widokiem, który możemy używać w innych miejscach aplikacji.

    Taki styl programowania jest obecnie bardzo popularny m.in. w bibliotece React. Każdy kawałek aplikacji traktuje sie jako komponent: zaczynając od facebookowego przycisku 'Like' kończąc na całej widocznej stronie.

    Jak widać w AngularJS jest to obiekt JavaScriptowy, będący konfiguracją. Definujemy kontroler, widok, jak również można zdefiniować wiązania (bindings), dzięki czemu inne komponenty mogą reagować na to co dzieje się danym komponencie.

    W przypadku powyższego komponentu, wywołujemy go wprost w HTML jako:
    <main-page></main-page>

  • Usługa
    angular.module('app')
    .service('resourceService', ['$http',
     function($http){
       this.getSomeResource = function() {
         return $http.get('http://some/resource/url');
       }
     }
    ])

    Usługa to w AngularJS obiekt wykonujący jakąś funkcjonalność nie związaną bezpośrednio z widokiem (w porównaniu do kontrolera), np: pobieranie danych z serwera, wykonywanie logiki biznesowej po stronie aplikacji.

    Są one singletonami: tworzone są przez Angular tylko raz i ta instancja użwana jest w całej aplikacji.

Łączenie widoku z kontrolerem

AngularJS jest bardzo rozbudowanym frameworkiem, dlatego moge omówić tylko najistotniejsze elementy. Ważnym aspektem frameworku MVC jest łączenie kontrolera z widokiem. AngularJS umożliwia umieszczanie specjalnych wyrażeń wprost w pliku HTML - zostaną one zastąpione przez framework podczas renderowania strony.

Podstawowym elementem wiązacym jest interpolacja - {{ }}. Wewnątrz podwójnych klamer możemy umieścić dowolne wyrażenie JavaScriptowe oraz wyświetlić zmienne przypisane do $scope kontrolera.

W celu prezentacji prostoty użycia AngularJS umieściłem poniżej przykład dynamicznej listy:

angular.module('app')
.controller('sampleController', ['$scope',
 function($scope){
   $scope.tplData = {
     inputText: {{tplData.inputText}}
     list: [
      {{item}}
      ]
   }
 }
])

  • Do wiązania wartości <input /> używa się specjalnego atrybutu ng-model="tplData.inputText"

  • Do wyświetlania listy używa się atrybutu ng-repeat. Jego użycie pokazałem poniżej:

    <span ng-repeat="item in tplData.list">
     <span>{{item}}</span>
    </span>

  • Do dodawania / usuwania użyłem odpowiednika onlick Angulara: ng-click
